## Что это

**ВЕЧЕ** — это мессенджер, который целиком помещается в одну папку и запускается командой `node server.js`. Никаких аккаунтов Google, никаких внешних API, никакой аналитики. Вы поднимаете его на своём компьютере или сервере — и получаете полноценный современный мессенджер: личные чаты, группы, каналы, голосовые, видео-кружки, звонки, ленту постов, и всё это работает прямо в браузере и ставится как приложение (PWA).

Название — не случайное. **Вече** — народное собрание в древней Руси, где решения принимались сообща и которое нельзя было «отменить сверху». Идея проекта та же: средство общения, которое принадлежит людям, а не платформе. Его можно скопировать, поднять заново, раздать друзьям — и не существует единой кнопки, которой его можно выключить.

> Это подарок обществу. Код открыт (лицензия AGPL-3.0). Берите, поднимайте, улучшайте, раздавайте.

## Возможности

**Общение**
- 💬 Личные чаты, 👥 группы, 📣 каналы (в каналах пишут админы — читают все)
- 📡 «Эфир» — общий чат для всех на инстансе
- ↩️ Ответы, ⤴️ пересылка, ✏️ редактирование, 🗑 удаление у всех, 📌 закрепление
- 😀 Реакции эмодзи, индикаторы «печатает…» и «стирает…», галочки прочтения
- 🔍 Поиск по людям и чатам

**Медиа**
- 🖼 Фото и альбомы (с автосжатием на клиенте), 📎 файлы до 40 МБ
- 🎤 Голосовые сообщения (Opus 96 kbps, волна + перемотка + скорость 1×/1.5×/2×)
- ⭕ Видео-кружки с камеры (как в Telegram), переключение фронтальной/основной
- 📍 Геолокация, 🔗 превью ссылок (Open Graph)

**Живое**
- 📞 Аудио-звонки 1-на-1 (WebRTC, работают и когда приложение закрыто — через push)
- 🎮 Игры в переписке: крестики-нолики, 4-в-ряд, камень-ножницы-бумага, кости
- 📊 Опросы «Сходка», ⏱ отложенная отправка, 🔥 исчезающие сообщения

**Лента**
- 📰 Общая лента постов (как в соцсети) с лайками и просмотрами
- 📖 Личная «стена» в профиле каждого

**Приватность и порядок**
- 🔒 Секретные чаты 1-на-1 со сквозным шифрованием текста (ECDH P-256 + AES-256-GCM на Web Crypto, ключ не покидает браузер) — включаются вручную замком; вложения в этом режиме заблокированы, группы/каналы/«Эфир» не шифруются. Подробности и ограничения — [ниже](#секретные-чаты)
- 👻 Режим «Призрак» — полная невидимость: вас не найти по нику, видят только знакомые
- 🔒 Закрытый профиль, скрытие статуса «в сети», свой кастомный статус
- 🚫 Блокировка, ⚠️ жалобы, антифлуд и фильтр рекламы/запрещёнки
- 🛡 Панель модерации для администратора (жалобы, баны, удаление сообщений)

**Технологии**
- 📱 PWA: ставится как приложение на телефон и десктоп, работает офлайн-оболочка
- 🔔 Web Push уведомления о сообщениях и звонках
- 🌗 Тёмная и светлая тема, «жидкое стекло» (Liquid Glass) в оформлении
- ✨ Опциональный ИИ-ассистент «Алиса» (YandexGPT): ответы, резюме чата, расшифровка голосовых, напоминания

## Быстрый старт

Нужен только **[Node.js](https://nodejs.org) 18+**.

```bash
git clone https://github.com/romashka5911anton-cmyk/veche.git
cd veche
npm install
npm start
```

Откройте **http://localhost:4040**. Первый зарегистрировавшийся становится администратором.
Чтобы зайти с телефона в той же Wi-Fi — используйте адрес `http://ВАШ-IP:4040`, который сервер печатает при запуске.

Готово. Никакой базы данных ставить не нужно — данные лежат в `data.json` рядом с сервером.

## Развёртывание на своём сервере

Чтобы ВЕЧЕ был доступен из интернета и работали микрофон/камера/звонки, нужен **HTTPS** (браузеры разрешают доступ к медиа только на защищённых доменах). Ниже — типовая схема: Node за nginx с бесплатным сертификатом Let's Encrypt.

**1. Запустите ВЕЧЕ как службу (systemd):**

```ini
# /etc/systemd/system/veche.service
[Unit]
Description=VECHE messenger
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/veche
ExecStart=/usr/bin/node server.js
Environment=PORT=4040
Restart=always
User=veche

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now veche
```

**2. Проксируйте через nginx (важно передать WebSocket-заголовки):**

```nginx
server {
    server_name veche.example.org;

    location / {
        proxy_pass http://127.0.0.1:4040;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;      # WebSocket
        proxy_set_header Connection "upgrade";       # WebSocket
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 45m;                    # под загрузку файлов до 40 МБ
    }
}
```

**3. Выпустите сертификат:** `sudo certbot --nginx -d veche.example.org`

**4. (Опционально) звонки через NAT** — поднимите TURN-сервер (coturn) и пропишите `TURN_HOST` / `TURN_SECRET` в `.env`. Без него звонки работают только при возможности прямого P2P.

Настройки — через файл `.env` (см. [`.env.example`](.env.example)) или переменные окружения.

## Как устроено (коротко)

```
veche/
├── server.js              # весь сервер: HTTP + Socket.IO + хранилище (один файл)
├── lib/
│   ├── games.js           # правила мини-игр (чистые функции)
│   └── moderation.js      # фильтр контента — правьте под свою площадку
├── public/                # клиент: чистый JS/HTML/CSS, без сборки
│   ├── index.html         # разметка приложения
│   ├── app.js             # вся логика клиента (SPA на ванильном JS)
│   ├── style.css          # оформление (Liquid Glass, темы)
│   ├── emoji.js           # набор эмодзи
│   └── sw.js              # service worker (PWA + push)
└── tools/gen-icons.js     # генератор иконок (без зависимостей)
```

- **Реальное время** — Socket.IO (WebSocket с фолбэком на long-polling).
- **Хранилище** — один JSON-файл `data.json`, целиком в памяти, с отложенной записью на диск. Просто, прозрачно, легко бэкапить и мигрировать. Для больших нагрузок это первое, что стоит заменить на настоящую БД — см. [ARCHITECTURE.md](ARCHITECTURE.md).
- **Клиент** — никакой сборки, никакого фреймворка. Открыл файл — прочитал. Это осознанный выбор: чтобы **любой** мог разобраться и допилить.

Полное описание модели данных, событий сокета и точек расширения — в **[ARCHITECTURE.md](ARCHITECTURE.md)**.

## Секретные чаты

В личной переписке 1-на-1 есть **сквозное (end-to-end) шифрование текста**. Оно не включено по умолчанию: откройте личный чат и нажмите кнопку-замок 🔒. После этого текст сообщений шифруется прямо в браузере, и сервер видит только непрозрачный блок шифра.

Как это работает: стандартный Web Crypto браузера — ECDH P-256, HKDF-SHA256, AES-256-GCM. Своей самописной криптографии в проекте нет. На каждое сообщение генерируется одноразовая пара ключей, а ключ сообщения заворачивается отдельно для каждого получателя. Ваш приватный ключ создаётся неизвлекаемым (`extractable=false`), лежит в IndexedDB браузера и на сервер не уходит — туда отправляется только публичный ключ. По зашифрованному сообщению сервер не запускает ни Алису, ни модерацию, а в push пишет просто «🔒 Зашифрованное сообщение». Нужен HTTPS (Web Crypto работает только в защищённом контексте).

**Код сверки.** В секретном чате можно открыть «код сверки» (safety number) — 15 групп по 5 цифр, посчитанных из пары ваших публичных ключей. Ключ собеседника клиент берёт из данных сервера, и предупреждения о его смене нет — поэтому единственная защита от подмены ключа сервером или посредником в том, чтобы **сравнить этот код с собеседником по другому каналу** (вживую, по телефону, в другом мессенджере). Совпал — посредника нет.

**Что важно знать до того, как на это положиться:**

- Шифруется **только текст**. Фото, файлы, голосовые и кружки не шифруются — в секретном режиме их отправка просто заблокирована в интерфейсе.
- Только личка 1-на-1. Группы, каналы, «Эфир» и обычные (не секретные) личные чаты хранятся на сервере открыто. Чат с ИИ-ассистентом зашифровать в принципе нельзя.
- Метаданные — кто, кому, когда, факт переписки, прочтения, реакции — сервер видит всегда.
- Нет прямой секретности (PFS) и ratchet: если приватный ключ получателя окажется скомпрометирован, раскроется вся прошлая переписка. Сообщения не подписываются — авторство утверждает только серверное поле «от кого».
- Секретный режим — локальный флаг вашего устройства: он не синхронизируется и не мешает собеседнику писать в этот же чат открытым текстом.
- Клиентский JS отдаёт тот же сервер. Это принципиальное ограничение любого веб-E2E: доверяя серверу код, вы доверяете ему многое.
- Независимого профессионального аудита криптографии **не было**.

> ⚠️ **Ключ привязан к одному браузеру.** Вход с нового устройства создаёт новый ключ и перезаписывает публичный на сервере — старую секретную переписку на новом устройстве расшифровать не получится. **Очистка данных браузера (или профиля/режим инкогнито) уничтожает ключ безвозвратно — вместе с доступом к вашей же секретной переписке.** Восстановить его нельзя ни вам, ни администратору.

## Безопасность — честно

ВЕЧЕ защищает пароли (scrypt), фильтрует загрузки от XSS, ограничивает перебор входа, имеет защиту от SSRF в превью ссылок и режим приватности «Призрак».

**Но главное, что нужно понимать: по умолчанию сообщения хранятся на сервере в открытом виде.** Единственное исключение — [секретные чаты 1-на-1](#секретные-чаты), где текст шифруется на устройствах; всё остальное (группы, каналы, «Эфир», обычные личные чаты, а также вложения даже в секретных чатах) владелец сервера технически может прочитать. Это нормально для «поднял для своих» и подходит для сообществ, доверяющих своему администратору, — но это надо понимать до того, как доверять ВЕЧЕ чувствительные тайны.

Что это значит на практике, какова модель угроз и где проходят границы сквозного шифрования — подробно и без прикрас в **[SECURITY.md](SECURITY.md)**.

## Можно ли запускать на нескольких серверах, чтобы «не забанить»?

Короткий ответ: **да — и это заложено в саму природу проекта.** Любой может поднять свой ВЕЧЕ; выключить их все разом невозможно. Это уже даёт устойчивость на уровне «много независимых островов».

Длинный ответ — про то, чем «много инстансов» отличается от настоящей **федерации** (когда серверы общаются между собой и сеть переживает падение любого узла), про anti-censorship стратегию (зеркала, домены, обфускация, работа через Tor) и про то, почему настоящая многосерверность требует end-to-end шифрования — в отдельном документе **[FEDERATION.md](FEDERATION.md)**.

## Технологии

| Слой | Что используется |
|------|------------------|
| Сервер | Node.js, Express, Socket.IO |
| Реальное время | WebSocket (Socket.IO) |
| Звонки | WebRTC (P2P), опциональный TURN (coturn) |
| Пуш | Web Push (VAPID) |
| Клиент | Ванильный JS + HTML + CSS (без сборки, без фреймворков) |
| Приложение | PWA (манифест + service worker) |
| Хранилище | JSON-файл (in-memory + запись на диск) |
| ИИ (опц.) | YandexGPT + SpeechKit |

Зависимостей всего три: `express`, `socket.io`, `web-push`.
